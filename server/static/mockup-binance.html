Known Limitations
Important to communicate
These are documented design decisions, not bugs — important to show buyers
⏰
Automated strategies only fire during US market hours (9:30am–4pm ET, Mon–Fri)
The engine checks Alpaca's market clock before running any strategy. Binance is 24/7 but will only receive signals when the US stock market is open. Manual API calls are unaffected. Fix: a future "always-on" mode for crypto-only accounts.
🖱
Manual Order Ticket doesn't support Binance yet
The Buy/Sell ticket on the Positions page routes to Alpaca regardless of which account is selected. A future PR will fix order routing to use the selected account's broker.
📊
Avg Entry Price & Unrealized P&L unavailable for Binance positions
The Binance spot balance API returns quantities only — no cost basis. Avg Entry Price shows as $0.00, Unrealized P&L shows as $0.00. Fix requires tracking entry price in our own DB per trade.
📉
Daily P&L and loss limit rules don't work for Binance accounts
Binance doesn't provide a "last equity" snapshot for intraday P&L calculation. Daily Loss Limit and Weekly Loss Limit risk rules always evaluate 0% for Binance accounts — they won't trigger. Fix requires storing our own equity baseline at session start.
ℹ
Binance Testnet keys reset periodically
Binance Testnet (paper mode) API keys are issued from testnet.binance.vision and are periodically invalidated. Users should regenerate keys if Testnet stops working. This is a Binance platform limitation.
